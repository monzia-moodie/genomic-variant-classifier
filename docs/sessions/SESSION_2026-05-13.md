## Lessons captured to memory rule 28

Three new sub-items appended to the apply-batch hygiene rule:

- **28(7)** PowerShell `return` at prompt scope is a no-op for control flow;
  use `throw` to actually halt, or wrap blocks in a function/script-block.
  `Write-Error` does NOT halt subsequent statements.
- **28(8)** Never delete a fallback artifact (e.g. `.partial` stub) until the
  replacement is verified on disk. Pattern: `if (Test-Path replacement) {
  Remove-Item fallback }`, not the inverse.
- **28(9)** For any artifact shipped via `present_files`, user must visually
  confirm the UI file card is present BEFORE running commands that assume the
  file on disk. Hash verification alone is insufficient when the file isn't
  there at all.
