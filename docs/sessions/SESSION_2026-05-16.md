## Lessons learned

1. **SCP destination vs launch script path**: When the repo is cloned into
   `/workspace/genomic-variant-classifier/`, the `outputs/` and `data/`
   directories are INSIDE the repo. But `launch_run10_vm.sh` hardcodes
   `SPLITS_DIR=/workspace/outputs/run9_ready/splits` (OUTSIDE the repo).
   Fix: symlink, or align SCP destinations to match launch script expectations.

2. **PowerShell here-string CRLF contamination is unfixable**: The
   `-replace "\`r\`n", "\`n"` workaround fails intermittently. The ONLY
   reliable pattern for SSH from PowerShell is single-line commands with
   single quotes: `ssh ... 'command here'`. Never use here-strings or
   semicolons in SSH commands from PowerShell.

3. **`wc -l` returns 0 on \r-only files**: KAN's progress bars use `\r`
   (carriage return) overwriting. When captured to a log file, these produce
   a file with content but zero `\n` characters. `tail -c N` works; `tail -N`
   and `wc -l` do not.

4. **meta_val.parquet and meta_test.parquet are required**: The original SCP
   plan only included 6 split files (X/y for train/val/test). The
   `load_splits()` function requires 8 files including the two meta parquets.

5. **pykan must be explicitly installed**: The Vast.ai PyTorch image does not
   include pykan. Without it, KAN silently falls back to MLP.
