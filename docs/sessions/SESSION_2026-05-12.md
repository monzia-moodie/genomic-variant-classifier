### Failed
- Launch attempt 1: workflow-aware preflight bugs (ClinVar VCF, torch_geometric)
- Launch attempt 2: data SCP'd to /workspace/{data,outputs}/ but launch script 
  uses repo-relative paths
- Launch attempt 3: training script uses absolute `/workspace/outputs/...` 
  while preflight uses relative paths
- Launch attempt 4: `ln -s` placed symlink INSIDE existing dir instead of 
  replacing it
- Auto-destroy: vastai 1.0.12 added interactive `input()` confirmation, 
  fails under nohup with `OSError: Bad file descriptor`
- Save: `_CNN1D` defined inside `CNN1DClassifier._build_model.<locals>` is 
  not pickle-able, crashed `joblib.dump()`

### Fixed
- Workflow-aware preflight (commits 8a3785a + bd75ed5) — landed before launch
- Path mismatch — `mv` data into repo, symlink `/workspace/{data,outputs}` 
  → repo paths
- Symlink trap — `rm -rf` before `ln -s` when destination might be recreated 
  as directory
- Truncated launch_run9_vm.sh — restored via `git checkout HEAD --`

### Learned (standing rules added in this session)
1. Vast.ai SCP destinations must be repo-relative or include explicit symlink 
   step in runbook (cost: ~10 min debug)
2. `vastai destroy ≥1.0.12` is interactive; auto-destroy in scripts MUST pipe 
   `yes` or `echo y`
3. `ln -s` does NOT replace existing real directories; use `rm -rf` first when 
   destination may have been recreated between fix attempts
4. PowerShell strips inner `"..."` from ssh command args — use single quotes 
   ONLY inside ssh wrappers, never double quotes
5. **STOP putting bash code inside `ssh ... '<bash>'` from PowerShell.** Use 
   `@'...'@ | ssh ... bash -s` with `-replace "`r`n", "`n"` to strip CRLF
6. PowerShell `@'...'@` heredocs preserve `\r\n` line endings; trailing `\r` 
   on each line breaks commands using last token as filename. Always 
   `-replace "`r`n", "`n"` before piping to remote bash
7. Vast.ai 2026 PyTorch images auto-tmux + auto-activate `/venv/main`. SCP 
   destinations MUST be inside the cloned repo. Subprocess can still use 
   `/usr/local/bin/python` symlinks for non-activated calls
