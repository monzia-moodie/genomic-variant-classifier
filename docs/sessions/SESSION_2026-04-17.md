## Learned
- **Silent-zero fallbacks in connectors are bugs, not features.**
  `SpliceAIConnector(vcf_path=None)` was returning 0.0 for all
  variants with only a `debug`-level log, so it looked like stub mode.
  Future connectors should assert file existence at startup-time
  rather than silently returning defaults at runtime.
- **gsutil is deprecated and has separate credential state from
  gcloud.** Never use `gsutil` for authoritative checks on GCS
  contents; always `gcloud storage ls`. This cost a full session of
  archaeology before today because the 2026-04-09 INCIDENT was marked
  RESOLVED but wasn't verifiable in later sessions.
- **Dataclass fields can't be monkeypatched via
  `setattr(Class, "field", value)`.** The patch appears to work but
  has no effect on new instances. Patch instance methods or module
  constants instead.
- **Class-scoped `@pytest.fixture(autouse=True)` is the right hammer
  when disk I/O shows up across a whole test class.** Per-test
  monkeypatches create silent footguns: any new test added to the
  class later forgets the patch and reintroduces the slow behaviour.
- **Run scoped tests before full suites when iterating on fixes.** A
  20-minute suite with the failing test at the end is the worst
  feedback loop. `pytest path::Class::test -v --timeout=N` catches
  regressions in seconds.
- **PowerShell heredocs (`@'...'@ | Add-Content`) corrupt reliably
  when the content contains triple-quoted Python strings or literal
  `-m "..."` commit messages.** Use standalone `.py` files written
  via an editor or downloaded, then executed. Don't cram complex
  content into PowerShell heredocs.

## Standing rules — proposed additions
- **INCIDENT-resolution verification:** never mark an INCIDENT as
  `RESOLVED` without running `gcloud storage ls <exact-URL>` (or
  equivalent authoritative check for the target system) and pasting
  the output into the INCIDENT doc. The 2026-04-09 INCIDENT was
  actually correct — the file WAS in GCS — but this wasn't
  verifiable in subsequent sessions due to stale gsutil creds.
